# app-development
ich habe gute Neuigkeiten! Ich habe meine alte Arbeit an der Wera-App und der Steuerung wiedergefunden und den Quellcode jetzt auf GitHub veröffentlicht:  
[https://github.com/vi0lin/app-development](https://github.com/vi0lin/app-development)  

Ich bin echt froh darüber! Vielleicht können wir uns an dem ein oder anderen Teil orientieren. Wenn ich mich richtig erinnere, war das App-Layout schon ziemlich performant und hat gut auf verschiedenen Gerätegrößen funktioniert.  

Der Quellcode ist nicht fertig und enthält sensible Inhalte, die ich so weit wie möglich geschwärzt habe, um sie zu desensibilisieren. Falls dir beim Durchsehen etwas auffällt, das besser entfernt werden sollte, lass es mich bitte wissen, damit ich es komplett herausnehmen kann.  

Ich fasse mal die wichtigsten Punkte zusammen:  

Das Projekt besteht aus drei Hauptteilen:  
- **wera-app**  
- **wera-app-api**  
- **wera-steuerung**  

### wera-app  
- **App-Gerüst/Scaffold**: Enthält eine Seiten-Navigation (ähnlich wie bei Discord).  
  - **Admin-Build**: Zusätzliche Navigation auf der rechten Seite mit Buttons für API-Tests. (Man könnte hier verschiedene Navigationskonzepte umsetzen, falls die App skaliert.)  
  - **Prototyp-Features**:  
  - (Unvollständige) Implementierung von Lobpreisliedern mit serverseitigem Abruf und internem App-Caching.  
  - Ansätze für User-Rechte-Management.  
  - Pseudo-Artikel mit LazyLoader zum Laden und Zwischenspeichern bei App-Neustart.  
  - Push-Notifications und Firebase-Registrierung.  

### wera-app-api  
- Das Backend habe ich vor fünf Jahren in Python geschrieben – es gibt zig Varianten.  
- Ich habe mit Multithreading experimentiert und Command-Line-Interface-Operationen eingebaut. Mit Python 3.14 ist einiges davon (z. B. durch die Änderungen am Global Interpreter Lock) einfacher geworden.  
- Es gibt zahlreiche Ordner, teilweise als „unused“ oder „obsolete“ markiert, da ich keine Variante ausgelassen habe.  
- Es wurden Lösungsansätze wie WebSockets oder GPIO von Google getestet. (Davon ist nichts fertig implementiert.)

### wera-steuerung  
- Hier habe ich versucht, die Netzwerkanbindung mit Eugens Steuerung umzusetzen.  
- Es hat eine Zeit lang funktioniert, aber irgendwann gab es Probleme.  
- Die Steuerung von Events über WebSockets war schwierig. Nach ein paar lauffähigen Tests habe ich etwas umgestellt, was dann leider nicht mehr funktionierte.  

### Hinweise  
- An vielen Stellen tauchen Platzhalter wie `{API_DOMAIN}` auf, wo eine IP oder Domain eingetragen werden muss
- Es fehlen teilweise Bilder und secrets, die nicht veröffentlicht werden dürfen.
- Der Code ist aktuell vermutlich nicht lauffähig, da ich sensible Inhalte entfernt habe.  
- Der Quellcode ist unübersichtlich, enthält auskommentierte Zeilen und Ideenskizzen, an die ich aktuell gar nicht mehr denke. Ich habe auch einige Notizen entfernt.  

Vielleicht lässt sich trotzdem etwas Brauchbares daraus ziehen. Lass mich wissen, was du denkst, und ob dir etwas auffällt!  

# This Project Is Unfinished And Aimed To Test Out
This app is still a work in progress and primarily serves as
a learning tool to build the necessary expertise for developing a professional
application tailored to my church community. It also includes an unfinished API
backend developed in Python. The source code is currently disorganized and requires
a comprehensive overhaul, while only a limited set of features has been
implemented to date. The main objective was to establish a rough outline
of the app's architecture exploring the capabilities and packages offered by Dart and to
understand how straightforward it is to integrate them with any API. This
provides a solid foundation for a robust, maintainable, and easily deployable framework
in the future.

> [!WARNING]
> As previously noted, the project is currently non-functional and difficult to maintain due to significant amounts of unused code and multiple missing files. This results from extensive experimentation conducted during development to support learning. Additionally, certain code sections require integration with functional API endpoints, secrets, and authentication data to enhance functionality.
